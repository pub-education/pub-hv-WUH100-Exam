Process analysis

Much of what I write is from experience and even though I of course learned it from various sources it was a long time ago and impossible to remember where it all came from. However, things that I look up and take from books during the process now I will provide references to.

HTML layout

Overview

We will use two font stacks one serif based for headings and one sans-serif based for all other text. For header we use a stack consisting of Bree Serif from google fonts, Times New Roman, Times, and serif as a fallback to browser default serif font. For all other elements the fot sack looks like this: MuseoSans_500 loaded from local files from server, Verdana, Geneva, Tahoma, and sans-serif as a fallback to browser default sans-serif font.

Using the provided modeled layout we have three sections a header, main, and footer. So, we start with entering those using the new semantic elements from HTML5 header, main, and footer, wrapped in two div elements with classes page-wrapper and content-wrapper.

We will be using HTML5 because it's a standard which support is almost universal even in many older browsers, as is evident from caniuse.com [1]. The idea of using other designs than "1." for ordered list items was taken from an answer in stackoverflow.com [2]. The pages are also saved as .html files as opposed to .xhtml which means they will not be parsed by the XML parser but the DOM will be created by the HTML parser.

I have chosen to use the IEEE reference system as described in the IEEE Reference Guide [3].

Header

The header section can be divided into two main parts, logo and navigation. We separate those by putting them into a div and a nav element respectively so we can position them easier later on.

Main Section

The main section where all content will go can be seen as divided into two main areas. An upper helo image with an overlayed text section and the main content section containing no less than three columns since you wanted the middle to move to the top for smaller screens. So, we section this off with two div elements to start with and then we add three section elements to the lower div to use for the three columns.

Each of the section elements contains an image and a few paragraphs of text. We section these off with two div elements and put an image element in the top one and three p elements in the lower.

Footer

The footer is divided into three columns each containing a head and text. We section it off with three div elements and put a h3 element and an ul section in each to make the text align nicely when we make it into a list.

With this we are done with the initial markup of the home page and the header and footer. Header and footer will be copied to subsequent pages so that we will have consistency in the feel and look of the site. We cannot use JavaScript to change content as needed nor server-side code to assemble pages on request in this particular assignment. We might tweak it later on if we need to in order to make it behave and look the way we want.

The Stylesheet Development Process

Main setup

I tend to try to write my stylesheets in separate files that each belongs to a specific section or component of the webpage/site. E.g. page, navigation, footer and main content. Hence, the main style sheet is called styles.css where I link in the other stylesheets in order. It also links in a reset stylesheet that removes and resets all padding, margins, borders, etc and makes sure that we are using box-sizing border-box instead of the default content-box. Why? Because it's much easier to calculate the sizes of elements when content, padding, and borders are included in the size and we only have to accommodate for margins. Important to remember is that we first import the reset stylesheet, otherwise we will have problems since the styles are applied in the order they appear. However, since the requirement here is to contain all styling in one document I will instead section off that document into sections for header, main content, and footer. The styling for the Analysis page will however be in a separate document as will the fonts. I've tried to find the source of my reset.css but I can't find it.I've had the file a long time and can't remember where or when I got it, hence no reference.

We use css variables for the colors we want to use which makes it easier to change later on if we need to.

Header Section

Starting with sizing the logo image to 200px. We define the size in the stylesheet even though the width is set to "200" in the img tag, we do that so that the much larger image used will not disturb and create some unwanted visual effects during rendering. Then we make the containing header element a flexbox and chose space-between for the justify-content directive to put logo and navigation at each end of the box. Then we remvove the bullets in front of each item and set the li elements inline. Then we set their hovering response, using transition for a smoother response. To use more than one property in transitions see StackOverflow.com [4]. We have a link to each of the two pages and two dummy links linked to the home/index page.

Main Content Section

Next up is the main / content section. We start with the helo image and and put that in a new div element. Then we put the p element inside a new divs so that we can place it and give the p element a faded background for the text, making it stick out a little better over the image. Then it's time for the three little content cards with their images. They are already in separate section tags that we use here instead of div tags since HTML5 has made these new semantic elements available to us. First, we will make the containing div element a flexbox so we can place the different sections as we wish. Could just as well have done this with grids. I might do another version later using grids instead of flexbox just for fun. Flexbox is by default set to row so no real need for using flex-direction here [5]. The cards section themselves are divide into two sections, card-image and card-text, with div elements. We create the rounded effect of the image part with a border-radius set to 50% and we make the card-image container square with a little trick using clamp() [6]. Clamp has a decent support amongst most modern browsers from mid 2020 and onwards [7]. For this purpose I will be happy with that, for a wider audience in a production environment we would have had to include JavaScript support to accomplish the same objective. Here we make sure that the image rezises relative to the vierwport when the viewport gets smaller than 960px wide. It's set to 60% of the size of one third of the viewport width. To accomplish the overlay effect with the rounded image(s) sticking up over the helo image we use a negative margin-bottom value in the main-helo class of which will make the main-content class elements render above the helo image. OK we had to set their z-index too to layer them. Then we also but a border around the image using the same color as the background to get that upper frame effect even though it goes around the whole image.

Footer Section

We use flexbox here too for controlling the layout because it's pretty easy and we don't have to make any advanced stylings for it. We add a little padding and a margin at the top removes the bullets from list elements and a little background collor and we're done. While I was writing a comment for the footer element I remembered something I've used many times before using "mailto:" with emails. So I looked it up on MDN and also found "tel:" that I also included [8]. I found an explanation for tel: interesting for me since I've never used it, as far as I can remember, before even though it's old too at a blog [9].

Making the pages Responsive

We start by looking at what the requirements are. Here it says at least one breakpoint. So I think we set two. We already have one size defined and that is 960px which is the width of our content wrapper. So we don't need to bother about larger screens the content will only be 960px wide anyhow. Smaller sizes we take a look and see where things starts happening. It looks OK down to 650px so we set our first breakpoint there. This is only for the helo text. The rest can survive down to 600px where we put our second breakpoint which will be our main breakpoint towards mobile devices. The content should be OK down to a width of 300px.

I've chosen a different approach than the one suggested in the mobile layout provided and instead made the menu display as a list and kept the logo to the left and not centered as depictedin the model. To make the logo display on top of the menu you simply change the flex-direction to column and in this case the logo will be on top because of the order of the elements in the html document. It would be a nice touch to set the justify-conent to flex-start and the align-items attribute to center. If necessary I can re-write it using that if my solution isn't satisfactory. We could also have made the footer content lin-up on top of each other instead of changing font sizes, but we'll save that for another day.

Printing

For printing we don't need the helo text or image nor do we need the menu. The footer on the other hand might be useful. We set display none on both nav and the main-helo class so they will disappear from the rendering. We had to use !important on the nav to make that disappear but since it's only a simple change only for print I deem it acceptable. Some pointer and ideas came from Craig Buckler's artcle on sitepoint.com [10].
